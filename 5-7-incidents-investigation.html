Threat Hunting

// Built-in hunting queries available in Sentinel â†’ Hunting

// Example: Hunt for suspicious PowerShell
DeviceProcessEvents
| where TimeGenerated > ago(7d)
| where FileName == "powershell.exe"
| where ProcessCommandLine contains "-enc" 
    or ProcessCommandLine contains "-encoded"
    or ProcessCommandLine contains "downloadstring"
    or ProcessCommandLine contains "bypass"
| project TimeGenerated, DeviceName, AccountName, ProcessCommandLine

// Hunt for data exfiltration
DeviceNetworkEvents
| where TimeGenerated > ago(24h)
| where RemotePort in (443, 80)
| summarize 
    TotalBytes = sum(SentBytes),
    Connections = count()
    by DeviceName, RemoteIP, bin(TimeGenerated, 1h)
| where TotalBytes > 100000000 // 100 MB

// â†’ Add to existing incident or create new incident

Hunting Workflow

HUNTING WORKFLOW:

1. HYPOTHESIS
   "Attackers may be using encoded PowerShell for C2"

2. QUERY
   Run hunting query across relevant data

3. ANALYZE
   Review results, identify suspicious patterns

4. BOOKMARK
   Save interesting findings as bookmarks

5. INVESTIGATE
   Create incident or add to existing incident

6. DOCUMENT
   Record findings and update threat intel

Querying Incidents

// Get all incidents
SecurityIncident
| where TimeGenerated > ago(30d)
| project TimeGenerated, IncidentNumber, Title, Severity, Status, Owner

// Open high severity incidents
SecurityIncident
| where Status == "Active"
| where Severity == "High"
| project IncidentNumber, Title, Owner, CreatedTime

// Incident metrics
SecurityIncident
| where TimeGenerated > ago(30d)
| summarize 
    Total = count(),
    TruePositives = countif(Classification == "TruePositive"),
    FalsePositives = countif(Classification == "FalsePositive")
| extend TPRate = round(TruePositives * 100.0 / Total, 2)

// Mean Time to Close
SecurityIncident
| where TimeGenerated > ago(30d)
| where Status == "Closed"
| extend MTTC = datetime_diff('hour', ClosedTime, CreatedTime)
| summarize AvgMTTC = avg(MTTC), MedianMTTC = percentile(MTTC, 50)

// Incidents by analytics rule
SecurityIncident
| where TimeGenerated > ago(7d)
| mv-expand AlertId = AlertIds
| join kind=inner (
    SecurityAlert
    | project AlertId = SystemAlertId, AlertName
) on AlertId
| summarize IncidentCount = dcount(IncidentNumber) by AlertName
| sort by IncidentCount desc

ğŸ’¼ Interview Question

"Walk me through how you would investigate an incident in Microsoft Sentinel."

1. Review incident details: Check severity, title, description, and mapped MITRE techniques
2. Examine alerts: Understand what triggered the incident and review the raw events
3. Use investigation graph: Visualize relationships between entities (users, IPs, hosts)
4. Check entity pages: Review user/host history, risk score, and related incidents
5. Query related data: Search for additional context in logs
6. Bookmark findings: Save important evidence
7. Take action: Run playbooks or coordinate manual response
8. Document and close: Add comments and select appropriate classification
