Key Security Components

Component Function Security Relevance
LSASS (lsass.exe) Local Security Authority Subsystem Handles authentication, stores credentials in memory - prime target for credential dumping (Mimikatz)
SAM Database Security Account Manager Stores local user account hashes - target for offline password attacks
Security Event Log Audit trail for security events Primary source for authentication, privilege use, and policy change events
Windows Registry Hierarchical configuration database Persistence mechanism, stores system/user configuration, malware often modifies Run keys
Services (services.exe) Service Control Manager Manages Windows services - attackers create malicious services for persistence
WMI (wmiprvse.exe) Windows Management Instrumentation Used for lateral movement, persistence (WMI subscriptions), and reconnaissance

Authentication Events (Logon/Logoff)

Event ID Description Key Fields Detection Use Cases
4624 Successful logon LogonType, TargetUserName, IpAddress, WorkstationName Lateral movement, unusual logon times, service account abuse

4634/4647 Logoff TargetUserName, LogonId Session duration analysis, correlate with 4624
4648 Explicit credential logon (RunAs) SubjectUserName, TargetUserName, TargetServerName Credential abuse, privilege escalation attempts
4672 Special privileges assigned (admin logon) SubjectUserName, PrivilegeList Admin activity tracking, privilege escalation

"46-24-25-88-98" - Remember these as pairs:

  • 4688 = Process creation (think "68" = process started)

Detection Queries

Splunk - Registry Run Key Modifications:

index=windows sourcetype=WinRegistry 
(registry_path="*\\CurrentVersion\\Run*" OR registry_path="*\\CurrentVersion\\RunOnce*")
| stats count by registry_path, registry_value_name, registry_value_data, host, user
| where NOT match(registry_value_data, "(?i)(microsoft|windows|program files)")

Sentinel KQL - Registry Persistence:

DeviceRegistryEvents
| where ActionType == "RegistryValueSet"
| where RegistryKey has_any (@"\CurrentVersion\Run", @"\CurrentVersion\RunOnce", 
    @"Winlogon\Shell", @"Winlogon\Userinit", @"Image File Execution Options")
| where RegistryValueData !contains "Microsoft" and RegistryValueData !contains "Windows"
| project TimeGenerated, DeviceName, InitiatingProcessAccountName, 
    RegistryKey, RegistryValueName, RegistryValueData

Essential PowerShell Commands for Security

System Information Gathering

# Get system information
Get-ComputerInfo | Select-Object WindowsProductName, WindowsVersion, OsArchitecture

# Get logged-on users
Get-WmiObject -Class Win32_ComputerSystem | Select-Object UserName
query user

# Get running processes with command line
Get-WmiObject Win32_Process | Select-Object ProcessId, Name, CommandLine

# Get network connections with owning process
Get-NetTCPConnection | Where-Object State -eq 'Established' | 
    Select-Object LocalAddress, LocalPort, RemoteAddress, RemotePort, OwningProcess, 
    @{Name='ProcessName';Expression={(Get-Process -Id $_.OwningProcess).Name}}

# Get services
Get-Service | Where-Object Status -eq 'Running' | Select-Object Name, DisplayName, StartType

Security Event Log Queries

# Get recent failed logons
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4625} -MaxEvents 50 |
    Select-Object TimeCreated, 
    @{N='User';E={$_.Properties[5].Value}},
    @{N='SourceIP';E={$_.Properties[19].Value}},
    @{N='Status';E={$_.Properties[7].Value}}

# Get recent successful logons (Type 10 = RDP)
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4624} -MaxEvents 100 |
    Where-Object {$_.Properties[8].Value -eq 10} |
    Select-Object TimeCreated,
    @{N='User';E={$_.Properties[5].Value}},
    @{N='SourceIP';E={$_.Properties[18].Value}}

# Get process creation events
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4688} -MaxEvents 50 |
    Select-Object TimeCreated,
    @{N='User';E={$_.Properties[1].Value}},
    @{N='Process';E={$_.Properties[5].Value}},
    @{N='CommandLine';E={$_.Properties[8].Value}}

# Check if security log was cleared
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=1102} -ErrorAction SilentlyContinue

Persistence Investigation

# Get startup programs (Run keys)
Get-ItemProperty -Path 'HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run'
Get-ItemProperty -Path 'HKCU:\SOFTWARE\Microsoft\Windows\CurrentVersion\Run'

# Get scheduled tasks
Get-ScheduledTask | Where-Object State -eq 'Ready' | 
    Select-Object TaskName, TaskPath, @{N='Action';E={$_.Actions.Execute}}

# Get services with unusual paths
Get-WmiObject Win32_Service | 
    Where-Object {$_.PathName -notlike '*System32*' -and $_.PathName -notlike '*Program Files*'} |
    Select-Object Name, State, PathName, StartMode

# Get WMI subscriptions (persistence)
Get-WMIObject -Namespace root\Subscription -Class __EventFilter
Get-WMIObject -Namespace root\Subscription -Class __EventConsumer
Get-WMIObject -Namespace root\Subscription -Class __FilterToConsumerBinding

Network Investigation

# Get all network connections
netstat -ano | findstr ESTABLISHED
Get-NetTCPConnection | Where-Object {$_.State -eq "Established"}

# Get DNS cache
Get-DnsClientCache | Select-Object Entry, RecordName, Data

# Get ARP table
Get-NetNeighbor | Where-Object State -eq 'Reachable'

# Get listening ports
Get-NetTCPConnection -State Listen | Select-Object LocalPort, OwningProcess,
    @{N='Process';E={(Get-Process -Id $_.OwningProcess).Name}}

"What Windows Event IDs would you use to detect lateral movement?"

Focus on these events:

  • 4624 (LogonType 3, 10) - Network logons and RDP, check for unusual source IPs
  • 4648 - Explicit credential usage (RunAs), often used with stolen creds
  • 4672 - Admin logons, track privileged access
  • 5140/5145 - Network share access, attackers access C$, ADMIN$
  • 7045 - New service installed (PsExec creates PSEXESVC)
  • 4688 - Process creation with network tools (PsExec, wmic, PowerShell remoting)

Correlate source IPs across these events to identify lateral movement paths.

Chapter Summary

  • Windows security relies on LSASS for authentication, SAM for local accounts, and Event Logs for auditing
  • Critical Event IDs: 4624/4625 (logon), 4688 (process), 4698 (task), 4720/4732 (accounts), 1102 (log cleared)
  • Logon Types indicate HOW authentication occurred: Type 3 (network), Type 10 (RDP)
  • Sysmon provides superior visibility: Event 1 (process+parent+hash), Event 10 (LSASS access), Event 3 (network per process)
  • Registry persistence: Run keys, Services, Winlogon, Image File Execution Options
  • PowerShell: Get-WinEvent for log queries, Get-ItemProperty for registry, Get-NetTCPConnection for network
  • CMD: netstat, tasklist, wmic, schtasks for quick triage without PowerShell
