Solutions bundle related content for specific data sources or use cases.

Solution Contents Cost

Palo Alto PAN-OS 1 connector, 15 rules, 2 workbooks, 3 playbooks Free
SOC Process Framework Workbooks, hunting queries, watchlists Free

Installing a Solution

Azure CLI 
# List available solutions
az sentinel content list --workspace-name "sentinel-ws" --resource-group "rg-sentinel"

# Install solution via ARM template
az deployment group create \
  --resource-group "rg-sentinel" \
  --template-uri "https://raw.githubusercontent.com/Azure/Azure-Sentinel/master/Solutions/Microsoft365Defender/Package/mainTemplate.json" \
  --parameters workspace-name="sentinel-ws"

Standalone Content

Individual content items can be installed without full solution packages.

Analytics Rule Templates

Pre-built detection rules ready to enable. Review and customize thresholds before activation.

Location: Analytics → Rule templates

Workbook Templates

Visualization dashboards for security monitoring. Save as custom workbook to modify.

Location: Workbooks → Templates

Hunting Queries

KQL queries for proactive threat hunting. Run ad-hoc or schedule as analytics rules.

Location: Hunting → Queries

Playbook Templates

Logic App templates for automated response. Deploy and configure connections.

Location: Automation → Playbook templates

CI/CD Deployment

Automate content deployment using Azure DevOps or GitHub Actions.

YAML - GitHub Actions Workflow 
name: Deploy Sentinel Content

on:
  push:
    branches: [main]
    paths:
      - '.sentinel/**'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Azure Login
        uses: azure/login@v1
        with:
          creds: ${{ secrets.AZURE_CREDENTIALS }}
      
      - name: Deploy Analytics Rules
        uses: azure/arm-deploy@v1
        with:
          resourceGroupName: rg-sentinel
          template: .sentinel/AnalyticsRules/template.json
          parameters: workspaceName=sentinel-ws
      
      - name: Deploy Workbooks
        uses: azure/arm-deploy@v1
        with:
          resourceGroupName: rg-sentinel
          template: .sentinel/Workbooks/template.json
Best Practice

Always review analytics rules before enabling. Customize thresholds and exclusions for your environment to reduce false positives.
