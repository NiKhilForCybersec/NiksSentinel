KQL Fundamentals

Kusto Query Language basics for security analysis

üìò Chapter Overview

KQL (Kusto Query Language) is the query language for Azure Data Explorer, Log Analytics, and Microsoft Sentinel. This chapter covers KQL fundamentals including syntax, operators, filtering, aggregation, and common security query patterns.

KQL Basics

Query Structure

// KQL uses a pipe-based structure (like Unix)
TableName
| operator1
| operator2
| operator3

// Example: Get failed sign-ins from last 24 hours
SigninLogs
| where TimeGenerated > ago(24h)
| where ResultType != 0
| project TimeGenerated, UserPrincipalName, IPAddress, ResultDescription

KQL vs SPL Comparison

Operation KQL (Sentinel) SPL (Splunk)
Start query TableName index=name
Filter | where field == "value" field="value"
Select columns | project field1, field2 | table field1, field2
Aggregate | summarize count() by field | stats count by field
Sort | sort by field desc | sort -field
Limit | take 10 | head 10
Time filter | where TimeGenerated > ago(24h) earliest=-24h

Filtering with where

// Basic equality
SigninLogs
| where ResultType == 0

// Not equal
SigninLogs
| where ResultType != 0

// String contains (case-insensitive)
SigninLogs
| where UserPrincipalName contains "admin"

// String contains (case-sensitive)
SigninLogs
| where UserPrincipalName contains_cs "Admin"

// Starts with / ends with
SigninLogs
| where UserPrincipalName startswith "svc_"
| where UserPrincipalName endswith "@company.com"

// In list
SigninLogs
| where ResultType in (50053, 50126, 50074)

// Not in list
SigninLogs
| where AppDisplayName !in ("Azure Portal", "Microsoft Teams")

// Regex match
SigninLogs
| where UserPrincipalName matches regex @"admin\d+@"

// Multiple conditions (AND)
SigninLogs
| where ResultType != 0
| where TimeGenerated > ago(1h)
// OR use: | where ResultType != 0 and TimeGenerated > ago(1h)

// Multiple conditions (OR)
SigninLogs
| where ResultType == 50053 or ResultType == 50126

// NULL checks
SigninLogs
| where isnotempty(IPAddress)
| where isnotnull(DeviceDetail)

Time Operations

// Relative time
| where TimeGenerated > ago(24h)      // Last 24 hours
| where TimeGenerated > ago(7d)       // Last 7 days
| where TimeGenerated > ago(1h)       // Last 1 hour
| where TimeGenerated > ago(30m)      // Last 30 minutes

// Time range
| where TimeGenerated between (ago(7d) .. ago(1d))

// Absolute time
| where TimeGenerated > datetime(2024-01-01)
| where TimeGenerated between (datetime(2024-01-01) .. datetime(2024-01-31))

// Time bucketing (bin)
| summarize count() by bin(TimeGenerated, 1h)
| summarize count() by bin(TimeGenerated, 1d)

// Extract time components
| extend Hour = datetime_part("hour", TimeGenerated)
| extend DayOfWeek = dayofweek(TimeGenerated)
| extend WeekOfYear = week_of_year(TimeGenerated)

// Format time
| extend FormattedTime = format_datetime(TimeGenerated, "yyyy-MM-dd HH:mm")

Selecting and Renaming Columns

// Select specific columns
SigninLogs
| project TimeGenerated, UserPrincipalName, IPAddress, ResultType

// Rename columns
SigninLogs
| project 
    Time = TimeGenerated,
    User = UserPrincipalName,
    IP = IPAddress

// Add columns (keep all existing)
SigninLogs
| extend 
    IsSuccess = (ResultType == 0),
    Hour = datetime_part("hour", TimeGenerated)

// Remove columns
SigninLogs
| project-away OperationName, CorrelationId

// Reorder columns
SigninLogs
| project-reorder TimeGenerated, UserPrincipalName, *

Aggregation with summarize

// Count all
SigninLogs
| summarize count()

// Count by field
SigninLogs
| summarize count() by UserPrincipalName

// Multiple aggregations
SigninLogs
| summarize 
    TotalAttempts = count(),
    UniqueUsers = dcount(UserPrincipalName),
    UniqueIPs = dcount(IPAddress)

// Count by multiple fields
SigninLogs
| summarize count() by UserPrincipalName, ResultType

// Common aggregation functions
SigninLogs
| summarize 
    Count = count(),                    // Count rows
    DistinctCount = dcount(IPAddress),  // Unique count
    Sum = sum(BytesSent),               // Sum
    Avg = avg(DurationMs),              // Average
    Min = min(TimeGenerated),           // Minimum
    Max = max(TimeGenerated),           // Maximum
    FirstSeen = min(TimeGenerated),
    LastSeen = max(TimeGenerated),
    Values = make_set(IPAddress),       // Unique values as array
    AllValues = make_list(IPAddress)    // All values as array
    by UserPrincipalName

// Count with condition (countif)
SigninLogs
| summarize 
    TotalLogins = count(),
    FailedLogins = countif(ResultType != 0),
    SuccessLogins = countif(ResultType == 0)
    by UserPrincipalName

Sorting and Limiting

// Sort descending
SigninLogs
| summarize count() by UserPrincipalName
| sort by count_ desc

// Sort ascending
SigninLogs
| sort by TimeGenerated asc

// Sort by multiple columns
SigninLogs
| sort by UserPrincipalName asc, TimeGenerated desc

// Limit results
SigninLogs
| take 100

// Top N
SigninLogs
| summarize count() by UserPrincipalName
| top 10 by count_ desc

Security Query Examples

// Failed sign-ins summary
SigninLogs
| where TimeGenerated > ago(24h)
| where ResultType != 0
| summarize 
    FailedAttempts = count(),
    UniqueIPs = dcount(IPAddress),
    IPs = make_set(IPAddress)
    by UserPrincipalName
| where FailedAttempts > 10
| sort by FailedAttempts desc

// Successful logins from new IPs
let KnownIPs = SigninLogs
| where TimeGenerated between (ago(30d) .. ago(1d))
| where ResultType == 0
| distinct IPAddress;
SigninLogs
| where TimeGenerated > ago(24h)
| where ResultType == 0
| where IPAddress !in (KnownIPs)
| project TimeGenerated, UserPrincipalName, IPAddress, Location

// Windows security events - failed logins
SecurityEvent
| where TimeGenerated > ago(24h)
| where EventID == 4625
| summarize 
    FailedLogins = count(),
    TargetAccounts = make_set(TargetUserName)
    by IpAddress
| where FailedLogins > 10

// Azure Activity - resource deletions
AzureActivity
| where TimeGenerated > ago(7d)
| where OperationNameValue endswith "delete"
| project TimeGenerated, Caller, ResourceGroup, Resource, OperationNameValue

// Anomalous sign-in times
SigninLogs
| where TimeGenerated > ago(7d)
| where ResultType == 0
| extend Hour = datetime_part("hour", TimeGenerated)
| where Hour < 6 or Hour > 22
| project TimeGenerated, UserPrincipalName, IPAddress, Hour
üí° KQL Tips

Performance:
‚Ä¢ Filter early with where clauses
‚Ä¢ Use project to limit columns
‚Ä¢ Time filter first for best performance

Readability:
‚Ä¢ Use let statements for reusable variables
‚Ä¢ Comment with //
‚Ä¢ Use meaningful column aliases

üíº Interview Question

"Write a KQL query to find users with more than 5 failed logins in the last hour."

SigninLogs
| where TimeGenerated > ago(1h)
| where ResultType != 0
| summarize FailedCount = count() by UserPrincipalName, IPAddress
| where FailedCount > 5
| sort by FailedCount desc

üß† Memory Aid

KQL = "Query as a Sentence"

Read KQL like English:
SigninLogs | where Status == "Failed" | summarize count() by User
= "From SigninLogs, where Status is Failed, summarize count by User"

Key Operators to Remember:
‚Ä¢ where = "filter rows WHERE this is true"
‚Ä¢ project = "PROJECT only these columns onto screen"
‚Ä¢ extend = "EXTEND the table with new columns"
‚Ä¢ summarize = "SUMMARIZE into groups"

SPL ‚Üí KQL Quick Map:
stats count ‚Üí summarize count() | table ‚Üí project | eval ‚Üí extend | search ‚Üí where

Chapter Summary

  • KQL uses pipe-based syntax: Table | operator | operator
  • where filters rows; project selects columns
  • summarize aggregates data with functions like count(), dcount(), sum()
  • Time operations: ago(), between, bin()
  • extend adds calculated columns
  • Filter early and project only needed columns for performance
