SPL vs KQL

Query language comparison with side-by-side examples

üìò Chapter Overview

SPL (Search Processing Language) and KQL (Kusto Query Language) are the query languages for Splunk and Sentinel respectively. This chapter provides side-by-side comparisons to help you translate between platforms and prepare for interviews requiring both skills.

Syntax Basics

QUERY STRUCTURE COMPARISON:

SPL (Splunk):
index=security sourcetype=WinEventLog EventCode=4625
| stats count by src_ip
| sort -count
| head 10

KQL (Sentinel):
SecurityEvent
| where EventID == 4625
| summarize count() by IpAddress
| sort by count_ desc
| take 10

Filtering Data

BASIC FILTERING:

SPL:
index=security EventCode=4625
index=security EventCode!=0
index=security user="admin"
index=security user="admin" OR user="root"
index=security user="admin" src_ip="10.0.0.1"

KQL:
SecurityEvent | where EventID == 4625
SecurityEvent | where EventID != 0
SecurityEvent | where TargetUserName == "admin"
SecurityEvent | where TargetUserName == "admin" or TargetUserName == "root"
SecurityEvent | where TargetUserName == "admin" and IpAddress == "10.0.0.1"


STRING MATCHING:

SPL:
index=security user="*admin*"           // wildcard
index=security user IN ("admin","root") // list
index=security | regex user="^admin"    // regex

KQL:
SecurityEvent | where TargetUserName contains "admin"
SecurityEvent | where TargetUserName in ("admin", "root")
SecurityEvent | where TargetUserName matches regex "^admin"


TIME FILTERING:

SPL:
index=security earliest=-24h
index=security earliest=-7d@d latest=@d
index=security earliest="01/01/2024:00:00:00"

KQL:
SecurityEvent | where TimeGenerated > ago(24h)
SecurityEvent | where TimeGenerated > ago(7d) and TimeGenerated < startofday(now())
SecurityEvent | where TimeGenerated > datetime(2024-01-01)

Aggregation

COUNTING:

SPL:
| stats count
| stats count by user
| stats count by user, src_ip
| stats dc(user) as unique_users

KQL:
| summarize count()
| summarize count() by TargetUserName
| summarize count() by TargetUserName, IpAddress
| summarize unique_users = dcount(TargetUserName)


MULTIPLE AGGREGATIONS:

SPL:
| stats count, dc(src_ip) as unique_ips, 
        values(src_ip) as ips by user

KQL:
| summarize count(), 
        unique_ips = dcount(IpAddress),
        ips = make_set(IpAddress) by TargetUserName


TIME-BASED AGGREGATION:

SPL:
| timechart span=1h count by user
| timechart span=1d count

KQL:
| summarize count() by TargetUserName, bin(TimeGenerated, 1h) | render timechart
| summarize count() by bin(TimeGenerated, 1d) | render timechart


CONDITIONAL COUNTING:

SPL:
| stats count(eval(status="success")) as success,
        count(eval(status="failure")) as failure

KQL:
| summarize success = countif(ResultType == 0),
        failure = countif(ResultType != 0)

Data Transformations

FIELD SELECTION:

SPL:
| table _time, user, src_ip, action
| fields _time, user, src_ip
| fields - _raw, _time

KQL:
| project TimeGenerated, TargetUserName, IpAddress, Activity
| project TimeGenerated, TargetUserName, IpAddress
| project-away TimeGenerated, Type


FIELD RENAMING:

SPL:
| rename src_ip AS SourceIP, user AS Username
| eval NewField = OldField | fields - OldField

KQL:
| project SourceIP = IpAddress, Username = TargetUserName
| extend NewField = OldField | project-away OldField


CREATING FIELDS:

SPL:
| eval status = if(EventCode=4624, "success", "failure")
| eval hour = strftime(_time, "%H")
| eval upper_user = upper(user)

KQL:
| extend status = iff(EventID == 4624, "success", "failure")
| extend hour = datetime_part("hour", TimeGenerated)
| extend upper_user = toupper(TargetUserName)


STRING OPERATIONS:

SPL:
| eval domain = mvindex(split(user, "@"), 1)
| eval lower_user = lower(user)
| eval user_len = len(user)

KQL:
| extend domain = tostring(split(TargetUserName, "@")[1])
| extend lower_user = tolower(TargetUserName)
| extend user_len = strlen(TargetUserName)

Joins and Lookups

JOINS:

SPL:
index=auth 
| join type=inner src_ip [search index=threat | table src_ip, threat_score]

KQL:
SigninLogs
| join kind=inner (
    ThreatIntelligenceIndicator 
    | project IPAddress = NetworkIP, ThreatScore = ConfidenceScore
) on $left.IPAddress == $right.IPAddress


LOOKUPS:

SPL:
| lookup user_info.csv user OUTPUT department, title

KQL:
| join kind=leftouter (
    _GetWatchlist('user_info') | project UserPrincipalName, department, title
) on UserPrincipalName


SUBSEARCH / SUBQUERY:

SPL:
index=auth [search index=threat | table ip | rename ip as src_ip]

KQL:
let ThreatIPs = ThreatIntelligenceIndicator | distinct NetworkIP;
SigninLogs | where IPAddress in (ThreatIPs)

Security Query Examples

FAILED LOGINS BY USER:

SPL:
index=windows EventCode=4625
| stats count by user, src_ip
| where count > 10
| sort -count

KQL:
SecurityEvent
| where EventID == 4625
| summarize count() by TargetUserName, IpAddress
| where count_ > 10
| sort by count_ desc


BRUTE FORCE DETECTION:

SPL:
index=windows EventCode=4625
| bin _time span=10m
| stats count, dc(user) as targeted_users by src_ip, _time
| where count > 20 AND targeted_users > 5

KQL:
SecurityEvent
| where EventID == 4625
| summarize count(), targeted_users = dcount(TargetUserName) 
    by IpAddress, bin(TimeGenerated, 10m)
| where count_ > 20 and targeted_users > 5


PROCESS CREATION MONITORING:

SPL:
index=windows EventCode=4688
| where (process_name="powershell.exe" OR process_name="cmd.exe")
| where match(command_line, "(?i)(encoded|bypass|hidden)")
| table _time, host, user, process_name, command_line

KQL:
SecurityEvent
| where EventID == 4688
| where Process has_any ("powershell.exe", "cmd.exe")
| where CommandLine matches regex "(?i)(encoded|bypass|hidden)"
| project TimeGenerated, Computer, TargetUserName, Process, CommandLine


IMPOSSIBLE TRAVEL:

SPL:
index=azure sourcetype=azure:aad:signin
| sort 0 user, _time
| streamstats current=f last(location) as prev_location, 
              last(_time) as prev_time by user
| eval time_diff = _time - prev_time
| where time_diff < 7200 AND location != prev_location

KQL:
SigninLogs
| order by UserPrincipalName, TimeGenerated
| serialize
| extend prev_location = prev(Location), prev_time = prev(TimeGenerated),
         prev_user = prev(UserPrincipalName)
| where UserPrincipalName == prev_user
| extend time_diff = datetime_diff('minute', TimeGenerated, prev_time)
| where time_diff < 120 and Location != prev_location

üíº Interview Question

"Convert this SPL query to KQL: Find top 10 users with failed logins in the last 24 hours"

SPL:
index=windows EventCode=4625 earliest=-24h | stats count by user | sort -count | head 10

KQL:
SecurityEvent | where TimeGenerated > ago(24h) | where EventID == 4625 | summarize count() by TargetUserName | sort by count_ desc | take 10

Key translations: index‚Üítable, earliest‚Üíwhere TimeGenerated, stats count by‚Üísummarize count() by, sort -‚Üísort by desc, head‚Üítake

Chapter Summary

  • SPL uses index=name; KQL starts with TableName
  • SPL filters inline; KQL uses | where
  • stats ‚Üí summarize, eval ‚Üí extend, table ‚Üí project
  • sort -field ‚Üí sort by field desc
  • head N ‚Üí take N
  • Both support joins, regex, and time bucketing
