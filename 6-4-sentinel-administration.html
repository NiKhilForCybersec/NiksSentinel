Sentinel Administration

Health monitoring, cost management, and operational maintenance

üìò Chapter Overview

Effective Sentinel administration ensures the platform delivers value while controlling costs. This chapter covers health monitoring, cost optimization, content management, and operational best practices.

Health Monitoring

Data Connector Health

// Check connector status
SentinelHealth
| where TimeGenerated > ago(24h)
| where SentinelResourceType == "Data connector"
| summarize 
    LastStatus = arg_max(TimeGenerated, Status),
    ErrorCount = countif(Status == "Failure")
    by SentinelResourceName
| project SentinelResourceName, Status = LastStatus, ErrorCount

// Data ingestion by connector
Usage
| where TimeGenerated > ago(24h)
| summarize TotalGB = sum(Quantity) / 1000 by Solution
| sort by TotalGB desc

// Detect ingestion gaps
let ExpectedTables = dynamic(["SigninLogs", "SecurityEvent", "AzureActivity"]);
ExpectedTables
| mv-expand TableName = ExpectedTables to typeof(string)
| join kind=leftouter (
    union withsource=Table *
    | where TimeGenerated > ago(1h)
    | summarize LastEvent = max(TimeGenerated) by Table
) on $left.TableName == $right.Table
| extend MinutesSinceLastEvent = datetime_diff('minute', now(), LastEvent)
| where MinutesSinceLastEvent > 30 or isnull(LastEvent)

Analytics Rule Health

// Rule execution status
SentinelHealth
| where TimeGenerated > ago(24h)
| where SentinelResourceType == "Scheduled analytics rule"
| summarize 
    Runs = count(),
    Failures = countif(Status == "Failure"),
    LastRun = max(TimeGenerated)
    by SentinelResourceName
| extend FailureRate = round(Failures * 100.0 / Runs, 2)
| where FailureRate > 10
| sort by FailureRate desc

// Rule with no incidents (may need review)
let RulesWithIncidents = SecurityIncident
| where TimeGenerated > ago(30d)
| mv-expand AlertId = AlertIds
| join kind=inner (SecurityAlert | project AlertId = SystemAlertId, AlertName) on AlertId
| distinct AlertName;
resources
| where type == "microsoft.securityinsights/alertrules"
| where properties.enabled == true
| extend RuleName = name
| where RuleName !in (RulesWithIncidents)

Cost Management

// Daily ingestion trend
Usage
| where TimeGenerated > ago(30d)
| summarize DailyGB = sum(Quantity) / 1000 by bin(TimeGenerated, 1d)
| render timechart

// Cost by data type
Usage
| where TimeGenerated > ago(30d)
| where IsBillable == true
| summarize BillableGB = sum(Quantity) / 1000 by DataType
| sort by BillableGB desc
| take 10

// Identify high-volume sources
Usage
| where TimeGenerated > ago(7d)
| summarize GB = sum(Quantity) / 1000 by DataType, Computer
| top 20 by GB

// Free vs billable data
Usage
| where TimeGenerated > ago(30d)
| summarize 
    BillableGB = sumif(Quantity, IsBillable == true) / 1000,
    FreeGB = sumif(Quantity, IsBillable == false) / 1000
| extend TotalGB = BillableGB + FreeGB
| extend BillablePct = round(BillableGB * 100 / TotalGB, 1)

1. DATA COLLECTION RULES (DCR)
   ‚Ä¢ Filter events at source

2. BASIC LOGS
   ‚Ä¢ Use for verbose, rarely queried data

Export Analytics Rules

# Export all analytics rules via Azure CLI
az sentinel alert-rule list \
    --resource-group rg-sentinel \
    --workspace-name law-sentinel \
    --output json > analytics_rules_backup.json

# Export via REST API (PowerShell)
$rules = Invoke-AzRestMethod -Path "/subscriptions/{sub}/resourceGroups/{rg}/providers/Microsoft.OperationalInsights/workspaces/{ws}/providers/Microsoft.SecurityInsights/alertRules?api-version=2023-02-01" -Method GET
$rules.Content | Out-File "analytics_rules.json"

üíº Interview Question

"How do you ensure Sentinel is performing optimally?"

1. Health Monitoring: Check SentinelHealth table for connector and rule status, monitor data ingestion gaps
2. Cost Management: Track ingestion volume, use Basic Logs for verbose data, leverage commitment tiers
3. Detection Quality: Review false positive rates, tune rules, maintain MITRE coverage
4. Automation: Monitor playbook success rates, verify permissions
5. Regular Maintenance: Update content, refresh watchlists, review access controls
6. Documentation: Keep runbooks current, document customizations

  • Backup configurations via ARM templates and source control
