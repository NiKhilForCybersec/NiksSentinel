Manual IOC Addition

# Add IOC via search
| makeresults
| eval ip = "192.168.100.50"
| eval description = "Known C2 server"
| eval threat_group = "APT29"
| eval weight = 90
| eval _key = ip
| outputlookup append=true ip_intel

# Bulk upload from CSV
| inputlookup my_threat_list.csv
| rename malicious_ip AS ip
| eval weight = 80
| outputlookup append=true ip_intel

IOC Matching Searches

IP Intel Matching

# Match network traffic against IP threat intel
| tstats summariesonly=true count FROM datamodel=Network_Traffic 
  BY All_Traffic.src_ip, All_Traffic.dest_ip, _time span=1h
| `drop_dm_object_name("All_Traffic")`
| lookup ip_intel ip AS dest_ip OUTPUT description AS threat_description, weight
| where isnotnull(threat_description)
| eval risk_score = weight
| eval risk_message = "Connection to threat intel IP: ".dest_ip." (".threat_description.")"
| `map_notable_fields`

# Check if IP is in threat intel
| makeresults
| eval ip = "8.8.8.8"
| lookup ip_intel ip OUTPUT description, weight
| table ip, description, weight

Domain Intel Matching

# Match DNS queries against domain threat intel
| tstats summariesonly=true count FROM datamodel=Network_Resolution 
  BY DNS.query, DNS.src, _time span=1h
| `drop_dm_object_name("DNS")`
| lookup domain_intel domain AS query OUTPUT description, threat_group
| where isnotnull(description)
| `map_notable_fields`

# Extract domain from URL and match
index=proxy 
| rex field=url "https?://(?P[^/]+)"
| lookup domain_intel domain OUTPUT description, weight
| where isnotnull(description)

File Hash Matching

# Match file hashes from endpoint data
| tstats summariesonly=true count FROM datamodel=Endpoint.Filesystem 
  BY Filesystem.file_hash, Filesystem.file_name, Filesystem.dest
| `drop_dm_object_name("Filesystem")`
| lookup file_intel file_hash OUTPUT description AS threat_description, threat_group
| where isnotnull(threat_description)
| `map_notable_fields`

# Match process execution hashes
index=sysmon EventCode=1
| lookup file_intel file_hash AS SHA256 OUTPUT description, threat_group
| where isnotnull(description)
| table _time, Computer, User, Image, SHA256, description, threat_group

Threat Activity Dashboard

# Threat Intel Matches Over Time
index=* (tag=network OR tag=endpoint) earliest=-24h
| lookup ip_intel ip AS dest_ip OUTPUT description AS ip_threat
| lookup domain_intel domain OUTPUT description AS domain_threat
| lookup file_intel file_hash OUTPUT description AS file_threat
| eval threat_match = coalesce(ip_threat, domain_threat, file_threat)
| where isnotnull(threat_match)
| timechart span=1h count by threat_match

# Top Threat Sources (IPs)
index=* tag=network earliest=-24h
| lookup ip_intel ip AS dest_ip OUTPUT description, weight
| where isnotnull(description)
| stats count by dest_ip, description
| sort -count
| head 20

# Threat Intel Coverage Report
| inputlookup ip_intel | stats count AS ip_count
| appendcols [| inputlookup domain_intel | stats count AS domain_count]
| appendcols [| inputlookup file_intel | stats count AS file_count]
| appendcols [| inputlookup url_intel | stats count AS url_count]

# Threat Intel Freshness
| inputlookup ip_intel
| stats max(_time) AS latest_update, count by source
| eval age_days = round((now() - latest_update) / 86400, 1)

Threat Intelligence Best Practices

âœ“ TI Implementation Guidelines

Feed Selection
â–¡ Start with high-confidence, curated feeds
â–¡ Assign appropriate weights (confidence scores)
â–¡ Avoid low-quality, high-volume feeds initially
â–¡ Mix commercial and open-source feeds

IOC Management
â–¡ Set expiration dates for IOCs
â–¡ Remove stale indicators regularly
â–¡ Track false positive IOCs for removal
â–¡ Deduplicate across feeds

Detection Strategy
â–¡ Use TI matches as risk modifiers, not direct alerts
â–¡ Combine with other context (asset criticality, user behavior)
â–¡ Create tiered response based on confidence
â–¡ Investigate matches in context, not isolation

Handling False Positives

# Whitelist known good indicators
| inputlookup ip_intel
| lookup ip_whitelist ip OUTPUT whitelisted
| where isnull(whitelisted)
| outputlookup ip_intel

# Track TI false positives
index=notable rule_name="*Threat Intel*" disposition="False Positive" earliest=-30d
| stats count by threat_match_value
| sort -count

"How do you integrate threat intelligence into Splunk ES?"

1. Configure feeds: Set up TAXII, CSV, or API-based feeds in Configure â†’ Data Enrichment â†’ Threat Intelligence Downloads
2. Map to collections: IOCs go into KV store collections (ip_intel, domain_intel, file_intel)
3. Enable matching: Correlation searches use lookups to match events against IOCs
4. Tune responses: High-confidence matches create notables; lower confidence adds risk
5. Maintain hygiene: Set expiration, remove false positives, track feed freshness

Chapter Summary

  • Threat intel integrates external IOCs (IPs, domains, hashes, URLs)
  • IOCs stored in KV store collections per type
  • Feeds configured via TAXII, CSV, or custom inputs
  • Matching searches lookup events against intel collections
  • Assign weights (confidence scores) to feeds and IOCs
  • Use TI matches for risk scoring, not just direct alerting
  • Maintain feed hygiene: expiration, deduplication, whitelist
