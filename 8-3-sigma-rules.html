SIGMA Rules

Platform-agnostic detection format for portable security rules

ğŸ“˜ Chapter Overview

SIGMA is an open-source, generic signature format that allows you to describe detection rules in a platform-agnostic way. Rules can be converted to Splunk SPL, Sentinel KQL, and many other SIEM formats. This chapter covers SIGMA syntax, writing rules, and conversion workflows.

SIGMA Rule Structure

title: Mimikatz Command Line Detection
id: a642964e-bead-4bed-8910-1bb4d63e3b4a
status: test
description: Detects Mimikatz execution based on command line arguments
references:
    - https://github.com/gentilkiwi/mimikatz
author: Security Team
date: 2024/01/15
modified: 2024/03/01
tags:
    - attack.credential_access
    - attack.t1003.001
    - attack.s0002
logsource:
    category: process_creation
    product: windows
detection:
    selection:
        CommandLine|contains:
            - 'sekurlsa::'
            - 'kerberos::'
            - 'crypto::'
            - 'lsadump::'
            - 'privilege::debug'
    condition: selection
falsepositives:
    - Legitimate security testing
level: high

Rule Components

Field Required Description
title Yes Short descriptive name
id Yes Unique UUID
status Yes test, experimental, stable, deprecated
description Yes What the rule detects
logsource Yes Category, product, service
detection Yes Selection criteria and condition
level Yes informational, low, medium, high, critical
tags No MITRE ATT&CK and other tags
falsepositives No Known false positive sources

Detection Syntax

# BASIC MATCHING
detection:
    selection:
        EventID: 4625
    condition: selection

# MULTIPLE VALUES (OR)
detection:
    selection:
        EventID:
            - 4624
            - 4625
            - 4648
    condition: selection

# MULTIPLE FIELDS (AND)
detection:
    selection:
        EventID: 4688
        NewProcessName|endswith: '\powershell.exe'
    condition: selection

# MODIFIERS
detection:
    selection:
        CommandLine|contains: 'mimikatz'      # substring
        CommandLine|startswith: 'cmd /c'      # starts with
        CommandLine|endswith: '.ps1'          # ends with
        CommandLine|re: '.*encoded.*'         # regex
        CommandLine|contains|all:             # must contain all
            - '-enc'
            - '-nop'

# COMPLEX CONDITIONS
detection:
    selection1:
        EventID: 4688
    selection2:
        NewProcessName|endswith: '\powershell.exe'
    filter:
        User: 'SYSTEM'
    condition: selection1 and selection2 and not filter

Converting SIGMA Rules

# Install pySigma
pip install pysigma pysigma-backend-splunk pysigma-backend-microsoft365defender

# Convert to Splunk
sigma convert -t splunk rule.yml

# Convert to Sentinel/KQL  
sigma convert -t microsoft365defender rule.yml

# Convert with pipeline (field mappings)
sigma convert -t splunk -p sysmon rule.yml

# Batch convert entire directory
sigma convert -t splunk rules/ -o output/

Writing Custom SIGMA Rules

# Example: Detect suspicious scheduled task creation
title: Suspicious Scheduled Task Creation
id: 8e58fda8-c1dc-47e0-8c30-94f87b19c367
status: test
description: |
    Detects scheduled task creation with suspicious 
    characteristics indicating potential persistence
references:
    - https://attack.mitre.org/techniques/T1053/005/
author: Your Name
date: 2024/01/15
tags:
    - attack.persistence
    - attack.t1053.005
logsource:
    product: windows
    service: security
detection:
    selection:
        EventID: 4698  # Scheduled task created
    suspicious_paths:
        TaskContent|contains:
            - '\AppData\Local\Temp\'
            - '\Users\Public\'
            - 'powershell'
            - 'cmd.exe /c'
    filter_legitimate:
        TaskContent|contains:
            - '\Microsoft\Windows\'
            - '\Google\Update\'
    condition: selection and suspicious_paths and not filter_legitimate
falsepositives:
    - Legitimate software installation
    - IT management tools
level: medium
